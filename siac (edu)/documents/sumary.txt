Visão geral: {
    A pasta siac(edu) é uma tentativa de organização do sistema em MVC.
}

Arquitetura e estrutura de pastas: {
    - controllers: 
    Esta pasta contém toda a lógica de controle da aplicação.

    - db:
    Aqui fica o arquivo que armazena o pool de conexão com o banco de dados 
    ou a biblioteca de acesso ao banco de dados que estamos usando.

    - documents:
    Arquivos .txt de documentação, dicionário de dados, resumo do sistema, etc...

    - middlewares:
    Arquivos para lidar com ações intermediárias entre as requisições e as respostas.

    - models:
    Pasta onde estão os modelos de dados e lógica de acesso ao banco de dados.

    - public:
    Pasta para arquivos estáticos, como CSS, JavaScript e imagens.

    - routes:
    Onde são definidas as rotas da sua aplicação.

    -views: 
    Esta pasta contém as páginas HTML do servidor e os arquivos de visualização.
}

Back-end:{
    banco de dados -> db -> models -> routes -> controllers
}

Front-end: {
    views -> public (css)
}

API (comunicação front com back-end): {
    public (js)
}

Fluxo completo: {
    banco de dados <--> db <--> models <--> routes <-->
}
